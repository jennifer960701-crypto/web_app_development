# 產品需求文件 (PRD) - 食譜收藏夾系統

## 1. 專案概述

### 背景與動機
在資訊爆炸的時代，人們往往在各個平台（如網誌、社群媒體、影片網站）看到喜歡的食譜，卻難以集中管理。當真正要下廚時，經常找不到當初儲存的食譜，或是無法快速回顧自己過去的料理筆記。為了解決食譜散落各處、難以搜尋與整理的痛點，本專案將打造一個專屬的食譜收藏夾系統，讓使用者能輕鬆管理料理靈感。

### 目標用戶
- 喜歡下廚的使用者
- 想整理料理靈感的上班族
- 飲食控制族群（需記錄食材與熱量）
- 食譜分享者

### 核心價值主張
打造一個集中化、結構化且易於檢索的數位食譜管理平台，讓使用者能快速儲存靈感、彈性分類，並能加上個人筆記，成為個人的專屬線上食譜書。

## 2. 功能需求

### 2.1 食譜收藏
- **使用者故事**：作為一名喜歡下廚的使用者，我希望能夠手動新增食譜或匯入網址，以便將看到的好食譜儲存下來。

### 2.2 分類與標籤管理
- **使用者故事**：作為一名想整理料理靈感的上班族，我希望能自訂分類與加入多個標籤（如：中式、快手菜、低脂），以便將食譜有條理地歸類。

### 2.3 食譜搜尋與篩選
- **使用者故事**：作為一名飲食控制族群，我希望能夠透過關鍵字、分類或烹飪時間來搜尋與篩選食譜，以便快速找到適合今天烹調的料理。

### 2.4 食譜編輯與個人筆記
- **使用者故事**：作為一名食譜分享者，我希望能調整食譜內容並添加個人心得筆記（例如：減糖、多加香料），以便記錄每次烹調的改良經驗。

### 2.5 收藏夾管理
- **使用者故事**：作為一名經常下廚的使用者，我希望能建立多個收藏清單並調整排序（例如：本週菜單、節慶料理），以便根據不同情境快速取用食譜。

## 3. 非功能需求

### 技術限制
- 後端框架：Flask
- 模板引擎：Jinja2
- 資料庫：SQLite

### 效能與安全考量
- **效能**：系統操作（如新增食譜、搜尋）應於合理時間內回應，確保流暢的操作體驗。
- **安全**：需防範常見的 Web 安全漏洞（如 SQL Injection、XSS），保護使用者儲存的資料。

## 4. MVP 範圍 (Minimum Viable Product)

### Must Have (必要功能)
- 食譜收藏（手動新增食譜內容：標題、食材、步驟）
- 基本分類管理（新增/編輯/刪除分類）
- 食譜列表與檢視

### Should Have (重要功能)
- 網址匯入食譜（解析基本資訊如標題、網址連結）
- 標籤功能與食譜搜尋（關鍵字搜尋）
- 食譜編輯與個人筆記欄位

### Nice to Have (加分功能)
- 建立多個收藏清單並調整排序
- 進階篩選（依烹飪時間等）
- 響應式網頁設計 (RWD)，支援手機端瀏覽
